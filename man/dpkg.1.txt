WWAARRNNIINNGG
       This  manual is intended for users wishing to understand ddppkkgg's command
       line options and package states in more detail than  that  provided  by
       ddppkkgg ----hheellpp.

       It  should _n_o_t be used by package maintainers wishing to understand how
       ddppkkgg will install their packages. The descriptions of  what  ddppkkgg  does
       when installing and removing packages are particularly inadequate.

DDEESSCCRRIIPPTTIIOONN
       ddppkkgg  is  a  tool to install, build, remove and manage Debian packages.
       The primary and more user-friendly front-end for ddppkkgg  is  aappttiittuuddee(1).
       ddppkkgg  itself  is controlled entirely via command line parameters, which
       consist of exactly one action and zero or  more  options.  The  action-
       parameter tells ddppkkgg what to do and options control the behavior of the
       action in some way.

       ddppkkgg can also be used as a front-end to ddppkkgg--ddeebb(1) and  ddppkkgg--qquueerryy(1).
       The list of supported actions can be found later on in the AACCTTIIOONNSS sec-
       tion. If any such action is encountered  ddppkkgg  just  runs  ddppkkgg--ddeebb  or
       ddppkkgg--qquueerryy with the parameters given to it, but no specific options are
       currently passed to them, to use any such option the back-ends need  to
       be called directly.

              _p_a_c_k_a_g_e_-_f_i_l_e must refer to a directory instead.

              11.. Extract the control files of the new package.

              22..  If  another version of the same package was installed before

              of the new package, because new files are written  at  the  same
              time old files are removed.

              66..  Configure the package. See ----ccoonnffiigguurree for detailed informa-
              tion about how this is done.
